# Оптимизация производительности PHP-кода

## Цель работы
Изучить методы измерения производительности PHP-кода, выявить неэффективные участки программы и оптимизировать их за счёт изменения алгоритма поиска.

---

## Задачи
- Запустить исходный «медленный» скрипт.
- Измерить время выполнения кода.
- Найти причину низкой производительности.
- Переписать алгоритм, используя более эффективный способ поиска.
- Сравнить результаты до и после оптимизации.

---

## Описание задания

В программе создаётся массив из **50 000 товаров**, каждый из которых содержит:

- `id` — идентификатор товара;
- `name` — название;
- `price` — цена.

Необходимо найти товар по `id`.

Реализованы два варианта поиска:

### 1. Медленный поиск (`searchProductSlow`)
Используется **линейный поиск** через цикл `foreach`.

```php
foreach ($products as $product) {
    if ($product['id'] == $searchId) {
        return $product;
    }
}
```
Сложность: O(N)
Недостаток: при каждом поиске перебирается весь массив.

 ## 2. Быстрый поиск (searchProductFast)

Используется ассоциативный массив, где:

- ключ = id
- значение = объект товара

Проверка выполняется через isset():

return isset($indexedProducts[$searchId])
    ? $indexedProducts[$searchId]
    : null;

Сложность: O(1)
Преимущество: доступ к элементу происходит мгновенно.

## Реализация бенчмарка

Для измерения производительности создана функция runBenchmark().

Она выполняет:

- фиксацию времени начала (microtime(true)),
- запуск тестируемой функции заданное количество раз,
- фиксацию времени окончания,
- расчёт:
- общего времени выполнения,
- среднего времени одного вызова.

## Вывод

В ходе работы была выполнена оптимизация алгоритма поиска в массиве.

Замена линейного поиска O(N) на поиск через ассоциативный массив O(1) позволила значительно повысить производительность программы.

Полученное ускорение составило более 400 раз, что показывает важность выбора правильного алгоритма при работе с большими объёмами данных.
